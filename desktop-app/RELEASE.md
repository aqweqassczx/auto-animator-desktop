# Release and Update Playbook

## 1) Prepare updater keys

Run once (locally):

```powershell
tauri signer generate -w ~/.tauri/myapp.key
```

Copy public key to:
- `src-tauri/tauri.conf.json` -> `tauri.updater.pubkey`

Keep private key in CI secret.

## 2) Build installer in CI

Required command:

```powershell
npm ci
powershell -ExecutionPolicy Bypass -File .\tools\build_pipeline_runner.ps1
npm run tauri:build
```

Artifacts:
- `src-tauri/target/release/bundle/msi/*.msi`
- bundled runtime `src-tauri/runtime/pipeline_runner.exe` is packed into MSI
- updater manifest `latest.json` (generated by Tauri updater flow)

## 3) Publish

Upload MSI + `latest.json` to GitHub Release of [aqweqassczx/auto-animator-desktop](https://github.com/aqweqassczx/auto-animator-desktop).
`src-tauri/tauri.conf.json` endpoint must point to this `latest.json`.

## 4) Client behavior

At startup app calls update check.
If update is available, UI shows banner and `Обновить` button.
On click:
- downloads package,
- applies update,
- asks user to restart.

## 5) Rollback

- Publish previous stable version as newest `latest.json`.
- Clients on next startup will get the stable build.
